What a CAN frame looks like on the wire

A CAN frame is, at heart, just a string of 1s and 0s. But on a real bus those bits are voltages on a twisted pair — and the rules for how those voltages combine are what make CAN robust, self-arbitrating, and self-acknowledging. Paste a bitstream below (or arrive here from the Frame Inspector's “Bitstream with Bit Stuffing” link) and see exactly what an oscilloscope would show.

Dominant vs recessive — why 0 always wins

A CAN transceiver does not drive the bus symmetrically. To send a dominant bit (logic 0) it actively pulls CAN_H up and CAN_L apart, creating a clear differential voltage. To send a recessive bit (logic 1) it simply lets go — termination resistors relax both lines back to ~2.5 V, so the differential collapses to ~0 V. The receiver decides by the difference, which rejects common-mode noise picked up equally on both twisted wires.

Wired-AND. Because dominant means “actively driven” and recessive means “released”, the bus is a logical AND of every node: if any node sends a dominant 0, the whole bus reads 0. A recessive bit only survives if every node leaves it recessive. This single property powers both acknowledgement and arbitration below.

The ACK slot — proof someone heard you

Look at the three logic traces in the scope at the ACK bit (highlighted in red). The transmitter sends the entire frame, but when it reaches the ACK slot it deliberately transmits recessive and listens. Every node that received the frame with a valid CRC slams the bus dominant for that one bit. Thanks to wired-AND, the transmitter reads back a dominant bit on RX — instant proof that at least one other node is alive and got the message.

  • Sender TX — drives the whole frame, but goes recessive at the ACK slot.
  • Receiver TX — stays recessive the entire frame except it pulls the ACK slot dominant.
  • Bus / RX — the wired-AND of the two. Identical to Sender TX everywhere except the ACK bit, which the receiver forced dominant.

No ACK? If no node acknowledges, the ACK slot stays recessive, the transmitter detects an ACK error, and it retransmits. A lone node on a bus with no listeners will retransmit forever.

What if two ECUs transmit at once? — Arbitration

CAN has no master and no bus scheduler — any node may start transmitting when the bus is idle. If two start in the same bit time, wired-AND resolves it without anyone losing data. Every transmitter watches the bus while it sends. The moment a node sends a recessive 1 but reads back a dominant 0, it knows someone with a higher-priority message is talking over it, so it backs off immediately and becomes a receiver. The winner never even notices — its bits sailed through untouched.

Lower numeric ID = more dominant bits early = higher priority. The sequence each node arbitrates with is the SOF start bit followed by the 11-bit ID, and it is bit-stuffed like any other part of the frame — so the demo below shows the actual bits on the wire, stuff bits included. Non-destructive. Unlike Ethernet collisions, no message is corrupted or lost. The high-priority frame is delivered with zero delay; the loser simply retries the instant the bus is free again. The 11 (or 29) ID bits are the only arbitration field, so the ID doubles as the message's priority.
