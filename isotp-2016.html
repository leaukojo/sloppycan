ISO 15765-2:2016 — Large Payload Extensions

Two escape sequences that extend classic ISO-TP beyond its original limits

What is this page about? Classic ISO-TP (ISO 15765-2, originally published 2003) supports messages up to 4 095 bytes and single frames up to 7 bytes. The 2016 revision introduced two escape sequences to handle larger payloads — primarily driven by the adoption of CAN FD (Flexible Data-Rate), which allows CAN frames up to 64 bytes. Neither extension changes the FC / CF handshake logic; only the header encoding of the first frame differs.

Extension 1 — CAN FD Single Frame escape (SF_DL = 0)

Classic SF encodes its length in the lower nibble of the PCI byte, which limits it to 15 — but only 7 bytes are ever valid on classic CAN (the rest of the byte is data). CAN FD frames can carry up to 64 bytes, so a new mechanism was needed to let a single CAN FD frame carry more than 7 bytes.

The solution: when the lower nibble is 0, the next byte carries the real length. This is called the SF escape sequence.

Classic CAN only: the SF escape is meaningless on classic 8-byte CAN — a payload larger than 7 bytes simply cannot fit in one frame. This extension is only used on CAN FD hardware.

Extension 2 — 32-bit First Frame escape (FF_DL = 0)

The classic FF header stores the total message length in a 12-bit field, capping it at 4 095 bytes. The 2016 revision repurposes the value 0x000 as an escape: when the 12-bit field is zero, the next four bytes carry the real length as a 32-bit big-endian unsigned integer. This extends the limit to ~4 GB (4 294 967 295 bytes).

Works on classic CAN too. Unlike the SF escape, the 32-bit FF escape is not limited to CAN FD. An 8-byte classic CAN frame can carry the 6-byte header (10 00 + 4 length bytes) with 2 data bytes, and the rest follows in normal Consecutive Frames. This is useful for large firmware image transfers over classic CAN.

How a receiver detects the escape sequence

A receiver reads the first two bytes of the FF:

  1. If bytes 0–1 form a value where the upper nibble is 0x1 and the lower 12 bits are non-zero → classic FF. Length = those 12 bits.
  2. If bytes 0–1 are exactly 10 00 → escape FF. Read the next 4 bytes as a 32-bit big-endian length. Data starts at byte 6.

Practical note: Most automotive ECUs on classic CAN buses still only implement the 2003 standard. The 32-bit escape and FD SF escape are primarily found on ECUs with CAN FD hardware (e.g., newer body control modules, gateway ECUs, ADAS controllers). Always check the ECU's communication specification before assuming escape sequence support.
